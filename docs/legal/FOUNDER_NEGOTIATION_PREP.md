# Founder Negotiation Prep

**Purpose:** meeting plan and script for negotiating founder equity with the sales founder and data architecture founder.

**Position:** technical founder proposes **55% / 25% / 20%** founder economics, with an optional **5% pool** either created later or carved out immediately as **52.25% / 23.75% / 19% / 5%**.

**Tone:** calm, fair, precise, and founder-aligned. The goal is not to "win" the conversation. The goal is to create a structure that everyone can defend when things get hard.

---

## 1. Desired Outcome

By the end of the meeting, aim to agree on:

| Topic | Target agreement |
|---|---|
| Founder split | **55% technical founder / 25% sales / 20% data architecture** |
| Pool | Either no pool now, or 5% carved out proportionally as **52.25 / 23.75 / 19 / 5** |
| Vesting | Reverse vesting for everyone; sales and data on 2 years with 1-year cliff |
| Technical founder credit | 30-40% of technical founder shares vested on day one for existing product/IP |
| Roles | Clear founder responsibilities and decision rights |
| Product wedge | Core contract management plus AI contract chatbot |
| Enterprise wedge | Contract management + AI Q&A + enterprise trust controls |
| Expansion | Drafting Studio and advanced drafting workflows stay as accessory/future modules |
| Revenue plan | Annual SaaS, paid pilots, onboarding/migration fees, usage-based AI/OCR add-ons |
| Next step | Swiss lawyer converts the agreed term sheet into legal documents |

Do not try to settle every legal detail in the meeting. Settle the commercial principles, then send them to counsel.

---

## 2. Meeting Agenda

Use a structured 75-90 minute meeting.

| Time | Topic | Goal |
|---:|---|---|
| 0-10 min | Shared ambition | Align on why the company can be valuable |
| 10-20 min | Current asset base | Acknowledge that the app/product/IP already exists |
| 20-35 min | Roles and expectations | Confirm what each founder will own |
| 35-50 min | Equity proposal | Present 55 / 25 / 20 and the pool logic |
| 50-65 min | Product and reward model | Explain core pricing, vesting, salary timing, commission, milestones, dividends, exit |
| 65-80 min | Objections and alternatives | Handle asks for more equity without emotional debate |
| 80-90 min | Close | Agree on next steps and legal drafting |

---

## 3. Opening Script

Start with alignment, not percentages.

> I want us to structure this properly from the beginning. I value both of you, and I want the equity to feel meaningful enough that we are all motivated. At the same time, we need to be precise because the app, codebase, architecture, AI pipeline, and infrastructure already exist. So this is not an idea-stage equal split. My proposal recognises the product already built, while giving both of you serious cofounder ownership for the value we create from here.

Then anchor clearly.

> My proposed founder economics are: 55% for me as technical founder and product creator, 25% for sales, and 20% for data architecture. If we want a 5% pool immediately, we cannot just add it on top because that makes 105%. We either create the pool later, or we carve it out proportionally now as 52.25%, 23.75%, 19%, and 5% pool.

Then align the commercial story.

> Commercially, I think we should sell the core first: contract management, repository, metadata, renewals, obligations, risk visibility, and the AI contract chatbot. Drafting Studio is valuable, but I would keep it as an accessory/future module so the first sales story is simple and easy for customers to understand.

---

## 4. Your Core Argument

Keep repeating this logic calmly:

| Principle | Explanation |
|---|---|
| The company is not starting from zero | The product and IP already exist because the technical founder built them |
| Sales and data are valuable | They deserve meaningful cofounder equity because revenue and enterprise data trust are critical |
| Future value should be earned | Vesting, milestones, commission, and salaries reward future execution |
| The cap table must add to 100% | 55 / 25 / 20 is valid; 55 / 25 / 20 plus 5% pool is not |
| Legal structure matters | Reverse vesting, IP assignment, and leaver clauses protect everyone |

The key sentence:

> I am not trying to minimise your value. I am trying to separate value already created from value we will create together.

### Enterprise Core Narrative

Use this if the discussion moves from equity into product and go-to-market:

> The first enterprise product should not be "everything we can build." It should be core contract management plus AI contract Q&A, wrapped in enterprise trust controls. Drafting Studio is important, but it is expansion after customers trust us with their contracts and use the repository/chatbot layer.

| Enterprise buyer concern | Meeting answer |
|---|---|
| Security/procurement | We need a trust pack: security overview, data flow, DPA, subprocessors, retention, AI data handling |
| Identity | Entra ID / OIDC SSO should be treated as Enterprise roadmap priority |
| Permissions | Contract-level permissions and confidential-contract controls are key enterprise requirements |
| Chatbot trust | Answers need source citations and exact clause evidence |
| Migration | Bulk import and migration are paid onboarding/Enterprise package items |
| Reporting | Renewals, obligations, risk, missing metadata, and processing dashboards justify management buy-in |

Meeting line:

> Enterprise pricing is justified by trust, permissions, audit, migration, reporting, and support, not just by the AI chatbot.

## 5. The Offer

### Offer A: Clean Founder Split

Use this if you want maximum simplicity.

| Founder | Equity |
|---|---:|
| Technical founder | **55%** |
| Sales founder | **25%** |
| Data architecture founder | **20%** |
| Pool | **0% now** |
| **Total** | **100%** |

Pool is created later by proportional dilution if needed for hires, advisors, or milestones.

### Offer B: Pool From Day One

Use this if they insist on a 5% pool immediately.

| Holder | Equity |
|---|---:|
| Technical founder | **52.25%** |
| Sales founder | **23.75%** |
| Data architecture founder | **19%** |
| Pool | **5%** |
| **Total** | **100%** |

This preserves the relative economics of 55 / 25 / 20 but makes room for the pool.

### Your Preferred Position

Start with **Offer A**. If the pool matters immediately, move to **Offer B**. Do not add a 5% pool on top.

## 6. Expected Commitments

Ask each person to commit to specific ownership areas before finalising equity.

### Technical Founder

- Own product vision and technical roadmap.
- Own codebase, infrastructure, AI pipeline, reliability, and security.
- Lead technical delivery and architecture decisions.
- Support enterprise demos, technical due diligence, and implementation.

### Sales Founder

- Own ICP, outreach, demos, pipeline, proposals, pricing, negotiation, and closing.
- Deliver first paid pilots and first annual contracts.
- Build CRM discipline and repeatable sales process.
- Own measurable ARR targets.
- Build the enterprise sales pack with security/procurement answers, pricing one-pager, pilot offer, and objection handling.

### Data Architecture Founder

- Own data architecture, data governance, AI/data quality, RAG quality, and enterprise data story.
- Help pass security/compliance reviews.
- Improve customer trust in the AI/data model.
- Deliver production architecture, not only documents.
- Own the enterprise data trust narrative: tenant isolation, data flow, retention/deletion, quality metrics, and source grounding.

Ask directly:

> Are we each ready to be accountable for these areas for at least the next 2 years?

## 7. Reward Model To Present

Explain that equity is only one part of reward.

Before discussing compensation, anchor the product and pricing strategy:

| Commercial point | Position |
|---|---|
| Initial wedge | Core contract management plus AI contract chatbot |
| Primary buyer value | Visibility, search, renewals, obligations, risk, AI Q&A |
| Drafting Studio | Accessory/future module after core adoption |
| Target pricing | CHF 12k-24k/year Core Team, CHF 36k-72k/year Business, CHF 90k-180k+/year Enterprise |
| Paid pilot | CHF 7.5k-15k for 4-8 weeks, credited against annual contract if converted |
| Onboarding/migration | CHF 3k-50k depending on complexity |
| Enterprise justification | SSO, permissions, audit export, retention, DPA/security pack, migration support, SLA |

| Reward type | Technical founder | Sales founder | Data architecture founder |
|---|---|---|---|
| Equity | 55% founder economics | 25% founder economics | 20% founder economics |
| Day-one vesting | 30-40% of own shares for prior product/IP | 0% | 0% unless prior architecture was delivered |
| Future vesting | Monthly over 36-48 months | 2 years, 1-year cliff | 2 years, 1-year cliff |
| Salary | Low/deferred early; CTO salary when cash allows | Base/commission once revenue allows | Base/project compensation once revenue allows |
| Extra upside | Product/reliability bonus only when cash allows | Commission or milestone equity for closed ARR | Milestone equity for shipped data/AI architecture outcomes |
| Exit/dividends | By vested ownership | By vested ownership | By vested ownership |

Say:

> Equity rewards long-term company ownership. Commission rewards closed revenue. Milestones reward measurable delivery. Salary rewards ongoing work once the company can afford it.

## 8. Objection Handling

### If Sales Says: "I Need 30%"

Response:

> I understand why you anchor there because revenue is critical. But 25% is already very meaningful cofounder equity for a product that already exists. I am open to upside through commission or milestone equity tied to signed ARR. What I do not want to do is grant future sales value before it exists.

Counter options:

| Ask | Counter |
|---|---|
| Sales wants 30% | Keep 25%; add commission on closed first-year ARR |
| Sales wants more for pipeline | Pipeline does not count; signed and paid customers count |
| Sales wants CEO-level control | Discuss title/role separately from equity; require full-time commitment and measurable ARR targets |
| Sales brings signed customer before agreement | Consider milestone pool grant after payment is received |

Possible sentence:

> If you close the first CHF 100k ARR, I want you rewarded. But that should be a milestone reward, not free permanent equity on day one.

### If Data Architecture Says: "I Need 25%"

Response:

> I agree data architecture can be very important, especially for enterprise trust, AI quality, and compliance. But 20% is already strong cofounder ownership. If additional value is created through shipped architecture, governance, quality systems, or enterprise readiness, we can reward that through the pool.

Counter options:

| Ask | Counter |
|---|---|
| Data wants 25% | Keep 20%; add milestone grants for shipped data/AI governance outcomes |
| Data wants more for strategy | Strategy alone is not enough; production implementation matters |
| Data wants equal technical authority | Clarify that technical founder owns final product/code/infrastructure decisions |
| Data delivers major enterprise architecture before signing | Consider pool grant after delivery is accepted |

Possible sentence:

> I want the data role rewarded for production impact, customer trust, and enterprise readiness, not just for the title.

### If Both Push For Equal Thirds

Response:

> Equal thirds would be fair if we were three people starting with only an idea. That is not the situation. The product already exists. I am still giving both of you meaningful cofounder equity, but equal thirds would ignore the work and risk already taken.

Do not overexplain. Repeat calmly.

### If They Say: "But Future Sales/Data Is What Makes It Valuable"

Response:

> Yes, and that is why you both get substantial equity. But future contribution should vest and should be rewarded through real outcomes. The existing product should also be recognised.

## 9. What You Can Trade

Use these as negotiation levers instead of increasing base equity.

| Lever | When to use it |
|---|---|
| Sales commission | If sales wants more upside for revenue |
| Milestone pool | If either founder wants more for measurable outcomes |
| Title | If someone needs role recognition, but only if responsibilities match |
| Salary ramp | If cash becomes available after funding/revenue |
| Advisory support budget | If data/sales need tools or external support |
| Decision rights within role | Give autonomy inside their domain, not control over all company decisions |

Do not trade away:

- IP ownership clarity.
- Vesting.
- Leaver provisions.
- Technical founder product/code authority.
- Cap table math.
- The already-built product credit.

---

## 10. Red Lines

Do not accept:

| Red line | Why |
|---|---|
| Equal thirds | Ignores the already-built product |
| No vesting | Dangerous if someone leaves early |
| 55 / 25 / 20 plus 5% pool without adjustment | Adds to 105%, not a valid cap table |
| Sales commission plus high equity plus no targets | Over-rewards before value exists |
| Data equity increase based only on plans/slides | Rewards intention rather than shipped value |
| No IP assignment | Creates investor and ownership risk |
| No leaver clause | Creates dead-equity risk |
| Two minority founders can outvote technical founder on product/IP | Risky while the company depends on the product/IP already built |

---

## 11. Close The Meeting

Use this close:

> I think the fair structure is 55 / 25 / 20 as founder economics. If we need a 5% pool now, we carve it out proportionally. Everyone vests. I get partial day-one vesting for the product already built. You both get serious upside for the value you create from here. If we agree on this principle, the next step is to send it to a Swiss lawyer and convert it into the shareholders' agreement, vesting, IP assignment, and leaver clauses.

Ask for explicit agreement:

> Can we agree on this as the commercial principle before legal drafting?

If they are not ready:

> That is fine. Let us each write down what we think is fair, including the concrete commitments and risks behind it, and come back with numbers that add to 100%.
